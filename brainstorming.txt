Multi-master setup

Offline CDN nodes

Deferred materialization

4. Hybrid option
Keep symlinks for common, safe derivations (allow list).

Allow pattern-based derivations for trusted pipelines, possibly with rate limiting or quotas.

Upload webapp can still pre-generate most common deferred symlinks to avoid runtime contention.

5. Fully managed CDN scenario
Using symlinks as the allow list:

CDN can operate independently of the webapp.

Deferred transformations and materialization occur on-demand, entirely on the CDN.

No additional network protocol or coordination is needed.

FSCK + cleanup + csync2 keep all nodes consistent.

Conclusion: pre-generated deferred symlinks are the simplest and safest way to decouple CDN nodes from the webapp, maintain control over allowed transformations, and avoid runtime security/DoS issues. Pattern-based callbacks are possible for flexibility, but introduce complexity, risk, and possibly extra coordination.

You could start with symlinks and later add optional pattern-based derivations for advanced scenarios.

Perfect — that strongly favors pre-generated deferred symlinks rather than dynamic patterns:

Why symlinks work well here
Limited derivations

Only 4–5 scaling sizes → the total number of deferred symlinks per resource is very small.

No combinatorial explosion, so pre-generation is cheap and manageable.

Explicit control

Each allowed derivation is listed on disk.

CDN nodes cannot perform arbitrary transformations, eliminating DoS risk.

Fully decoupled CDN

Symlinks propagate automatically via csync2.

CDN nodes require no knowledge of the database or webapp logic.

Predictable cleanup

Daily FSCK / cleanup can safely remove derived resources without worrying about dynamically requested transformations.

Implementation simplicity
On upload: create symlinks for each of the 4–5 scaling sizes.

When a request comes in: follow symlink → materialize if missing → serve.

No need for runtime pattern matching, callbacks, or dynamic derivation logic on the CDN.

With such a small, fixed set of derivations, pre-generated deferred symlinks are the cleanest, safest, and simplest solution — fully decoupling the CDN from the webapp while retaining control and predictability.
